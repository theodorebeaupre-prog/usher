# AGENTS.md — working on usher

Guidance for AI agents (and humans) contributing to this repo.

## What this is

usher is a Go CLI that routes a coding task to the best installed AI-agent
CLI (claude, codex, gemini, opencode, amp, copilot, cursor, qwen) using the
user's existing subscriptions, then execs into that agent's native TUI — or
its print-and-exit mode under `-p`. Launcher, not wrapper. Read the
[README](README.md) first; it is the product contract.

## Architecture map

- `main.go` — flags, subcommands (doctor/list/version), interactive flow
  (`cmdLaunch`/`launchWithFallback`), headless flow (`cmdHeadless`, JSON
  envelope, timeout, stdin replay), color helpers (`useColor`/`sgr`).
- `internal/adapters/` — one file per agent (eight methods: Name, Bin,
  Detect, Installed, LaunchArgs, HeadlessArgs, QuotaError, Profile) +
  shared helpers + `testdata/quota/` fixtures with provenance README.
- `internal/router/` — PURE functions only (no I/O, clock, env):
  `Classify` (keyword buckets, priority order) and `Rank` (strength ×
  quota × pins; longest-prefix path pins, boundary-anchored).
- `internal/ledger/` — observed launch/quota events, JSON, atomic writes,
  never fails a launch (corrupt → fresh + Warning), per-vendor decay.
- `internal/launch/` — child process runner. `Opts{Timeout, Stdin,
  CaptureStdout}`; zero-value = TTY passthrough. Timeout kills the whole
  process group (unix) and returns (124, …, ErrTimeout).
- `internal/banner/` — embedded frames.json (generated by
  `assets/banner/frames.py`, the source of truth), 4-bit ANSI, batched
  runs, gated by TTY/NO_COLOR/USHER_NO_BANNER.

## Hard invariants (violating these fails review)

- Exactly one external dependency: BurntSushi/toml. No cobra, no charm.
- `gofmt -l .` empty and `go vet ./...` clean on every commit.
- Agent name strings and registry order exactly: claude, codex, gemini,
  opencode, amp, copilot, cursor, qwen (founding four, then alphabetical).
- Task types exactly: debug, review, refactor, test, docs, feature, other.
- Headless stdout purity: in `-p` mode stdout carries ONLY agent output
  (or only the JSON envelope with `--json`); usher's lines go to stderr.
- Failover only on non-zero exit + quota-pattern match; never on exit 0;
  never on timeout; never for a forced `--agent`.
- Ledger problems must never block a launch.
- 4-bit ANSI only, gated on `useColor()` (NO_COLOR compliance).
- README examples must match real output byte-for-byte — capture, don't
  invent. README.fr.md mirrors README.md: every docs change lands in both.

## Workflows

- Tests: `go test ./...` (unit + E2E). E2E pattern: build the real binary,
  fake agent shell scripts on an isolated PATH, XDG_CONFIG_HOME in a temp
  dir. Never let tests or demos launch a real agent CLI.
- Adding an adapter: copy an existing file, register in `All()`, extend the
  table tests, add a quota fixture (+ provenance note), verify flags
  against the CLI's real `--help` — reality wins over any plan/doc.
- Release: tag `vX.Y.Z` → GitHub Actions runs goreleaser → binaries +
  Homebrew cask to theodorebeaupre-prog/homebrew-tap (needs the
  HOMEBREW_TAP_GITHUB_TOKEN secret; `skip_upload: auto`).
- Demo recording: `assets/demo/usher-demo.tape` (vhs) — see
  `assets/demo/README.md`. Re-record rather than editing the GIF.
- Commit messages end with:
  `Co-Authored-By: Claude Fable 5 <noreply@anthropic.com>`

## Roadmap position

v0.3.1 shipped = feature-complete before 1.0. The 1.0 hardening pass:
interface freeze, shell completions, man page, adapters verified on real
machines, isatty-aware color, weight tuning from dogfooding. New features
beyond that need an explicit spec first (see docs/superpowers/specs/).
