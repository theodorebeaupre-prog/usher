# usher v0.2 — Design Spec

**Date:** 2026-07-11
**Status:** Approved (user: "Include it" — headless in, details delegated)
**Baseline:** v0.1.0 (main @ b31bf0f + skip-cask commit), shipped and public.

## Scope

Three pillars:

1. **Headless mode** — `usher -p "task"` routes to the best agent and runs that agent's own print-and-exit mode. Output passes through untouched; exit code propagates. On a quota error, usher **automatically fails over** to the next ranked agent (each installed agent tried at most once).
2. **Two new adapters** — GitHub Copilot CLI (`copilot`) and Cursor CLI (`cursor-agent`), riding the two biggest subscriptions v0.1 doesn't cover.
3. **v0.1 final-review follow-ups** — fallback/failover launches are recorded in the ledger; `doctor` prints the corrupt-ledger warning; quota-error patterns get grounded in captured fixtures where obtainable.

### Non-goals (v0.2)

`--json` envelope, `--timeout` guard, unified output schema, multi-account switching, LLM-assisted routing, daemon. Interactive mode behavior is unchanged except where noted.

## Headless mode

### CLI surface

- `-p` / `--print` flag (matching the convention users already know from `claude -p`).
- `usher -p "task"`: route → run winner's headless invocation → stream stdout/stderr through untouched → exit with the agent's exit code.
- Composes with existing flags: `--agent X -p` forces agent X (single attempt, **no failover** — a forced choice is the user's responsibility); `--why -p` prints the scoring table (stderr) before running.
- The banner NEVER appears in `-p` mode. Decision lines and failover notices go to **stderr** only — a script's stdout is pure agent output.

### Adapter interface change

`Adapter` gains one method:

```go
HeadlessArgs(prompt string) []string  // argv for print-and-exit mode; nil = headless unsupported
```

Planned invocations (each verified against the CLI's `--help` at implementation time — reality wins over this spec):

| agent | headless argv |
|---|---|
| claude | `-p <prompt>` |
| codex | `exec <prompt>` |
| gemini | `-p <prompt>` |
| opencode | `run <prompt>` |
| copilot | `-p <prompt>` |
| cursor | `-p <prompt>` |

An adapter returning nil from `HeadlessArgs` is excluded from the candidate pool **before** ranking in `-p` mode (scores reflect the actually-available pool). If no installed agent supports headless → error listing which agents would.

### Auto-failover (headless only)

On exit, the winner's `QuotaError(exit, stderrTail)` is checked exactly as in interactive mode. If it fires:

1. Record the quota event + save ledger (same ordering contract as v0.1: record before anything else).
2. Print one stderr line: `→ <agent> hit its cap — failing over to <next>`.
3. Run the next ranked headless-capable agent. Each agent is attempted at most once per invocation.
4. When the pool is exhausted, exit with the LAST agent's exit code plus a stderr summary line.

Every attempt (first + failovers) records a launch event in the ledger. Interactive mode's relaunch path gains the same launch-recording fix (v0.1 follow-up) but keeps its Y/n prompt.

Signals: same contract as v0.1 — SIGINT ignored in the parent while a child runs; Ctrl-C kills the child, usher exits with its code. No new signal handling.

## New adapters

Registry order: founding four first, then alphabetical — `claude, codex, gemini, opencode, copilot, cursor`.

**copilot** — bin `copilot`, name string `copilot`. InstallHint `npm install -g @github/copilot` (verify). Quota window 6h. Strengths: debug .80, feature .80, refactor .75, review .85, docs .75, test .85, other .75. Quota patterns (lowercase; refine against captured fixtures): `rate limit`, `quota`, `usage limit`, `premium request`.

**cursor** — bin `cursor-agent`, name string `cursor`. InstallHint `curl https://cursor.com/install -fsS | bash` (verify). Quota window 5h. Strengths: debug .80, feature .85, refactor .85, review .75, docs .70, test .80, other .80. Quota patterns: `rate limit`, `quota`, `usage limit`, `429`.

Both get the standard one-file adapter treatment: `Detect`/`Installed` via shared helpers, `LaunchArgs` (interactive) + `HeadlessArgs`, profile, tests. Interactive launch argv verified via `--help` like v0.1 Task 7 did.

## Polish follow-ups

1. **Ledger completeness:** `RecordLaunch` fires for every actual launch — interactive relaunches and headless failovers included.
2. **doctor:** prints `led.Warning` when set (one line, before the agent table).
3. **Fixtures:** quota patterns for all six agents live alongside a `testdata/quota/<agent>.txt` fixture file per agent where a real error string could be captured or sourced; tests assert patterns match their fixture. Where no real capture is obtainable, the fixture is the pattern list's documented best guess, marked as such in a comment.

## Testing

- **Adapters:** table tests for `HeadlessArgs` across all six; profile-completeness test extends automatically (it iterates `All()`); quota-fixture tests per agent.
- **E2E (fake agents):** `-p` passthrough (stdout is exactly the fake's output, exit code propagates); failover chain (fake A exits 1 with a quota string, fake B succeeds → stdout is B's output, ledger holds A:quota + A:launch + B:launch); forced `--agent A -p` does NOT fail over; no-headless-capable-agents error.
- **Regression:** all v0.1 suites must stay green; interactive path untouched except launch-recording.

## Risks

- **Vendor headless flags drift** — same mitigation as v0.1: verified at implementation against `--help`, one-file adapters, fixtures make breakage visible.
- **Copilot/Cursor CLIs not installed on the dev machine** — reality-check step degrades to documenting what could not be verified; patterns marked best-guess.
- **Failover loops** — structurally impossible: candidates list shrinks by one per attempt.
