# usher v0.3 — Design Spec

**Date:** 2026-07-12
**Status:** Approved (user: "do all of phase one" — details delegated)
**Baseline:** v0.2.0 (main @ bc8f89c), shipped and public.

v0.3 is the **last feature release before the 1.0 hardening freeze**: the roadmap's `--json` envelope and `--timeout` guard, the stdin-drain-on-failover fix, two more adapters, and the deferred record-launch-after-start polish.

## Scope

1. **`launch.Opts` refactor** — `launch.Run` grows an options struct enabling the three headless features without three ad-hoc signatures.
2. **`--timeout <duration>`** (headless only) — kill a hung agent, exit 124 (GNU timeout convention). No failover on timeout.
3. **stdin replay on failover** — piped stdin is buffered once and replayed to every attempt, fixing the v0.2 documented caveat.
4. **`--json`** (headless only) — machine-readable envelope on stdout; agent output captured into it rather than streamed.
5. **Adapters: Qwen Code (`qwen`) and Amp (`amp`)** — eight agents total.
6. **Polish** — launch events recorded only for attempts that actually started (v0.2 final-review note).

### Non-goals

LLM routing, multi-account, daemon, interactive-mode timeout, shell completions/man page (those belong to the 1.0 hardening phase).

## 1. launch.Opts

```go
type Opts struct {
	Timeout       time.Duration // 0 = none; on expiry the child is killed
	Stdin         io.Reader     // nil = os.Stdin (TTY passthrough)
	CaptureStdout bool          // true = stdout into the return value, not the terminal
}

func Run(bin string, args []string, dir string, opts Opts) (exitCode int, stdout string, stderrTail string, err error)
```

- Zero-value `Opts` reproduces v0.2 behavior exactly (interactive call sites pass `launch.Opts{}`; `stdout` return is "" when not capturing).
- Timeout via `exec.CommandContext`; on expiry return `(124, captured, tail, ErrTimeout)` where `ErrTimeout` is a package sentinel (`errors.Is`-able). SIGINT handling unchanged.
- Existing tests updated mechanically; new tests: timeout kills a `sleep`-ing child (≥124 path), stdin replay from a `strings.Reader`, capture mode returns stdout and writes nothing to the process stdout.

## 2. --timeout (headless only)

- `--timeout 90s` / `--timeout 2m` — parsed with `time.ParseDuration`; parse failure → usage error.
- Without `-p` → error `--timeout requires -p (headless mode)`.
- On timeout: stderr line `usher: <agent> timed out after <d>`, **no failover** (a hang is not a cap; the script decides), no quota event, exit **124**.

## 3. stdin replay

- In `cmdHeadless`, when stdin is not a char device: `io.ReadAll(os.Stdin)` once, then every attempt gets `Opts.Stdin = bytes.NewReader(buf)`.
- When stdin IS a TTY: `Opts.Stdin = nil` (passthrough, v0.2 behavior).
- README's stdin caveat sentence is replaced by the fix's description.

## 4. --json

- Requires `-p`; `--json` without `-p` → error.
- Captures the final agent's stdout (`Opts.CaptureStdout = true`); stderr still streams live (+tail). usher's own lines stay on stderr.
- On completion (success, exhaustion, or timeout) print ONE JSON object to stdout:

```json
{
  "agent": "codex",
  "task_type": "debug",
  "exit_code": 0,
  "duration_ms": 4210,
  "output": "Patched the nil-check…",
  "attempts": [
    {"agent": "claude", "exit_code": 1, "quota_error": true, "timed_out": false, "duration_ms": 800},
    {"agent": "codex", "exit_code": 0, "quota_error": false, "timed_out": false, "duration_ms": 3410}
  ]
}
```

- Top-level fields describe the FINAL attempt; `attempts` lists all, in order. `output` is the final attempt's captured stdout (empty on total failure). Marshal with `encoding/json`; exit code of usher = final attempt's exit code, exactly as without `--json`.

## 5. New adapters

Registry order (founding four, then alphabetical additions): `claude, codex, gemini, opencode, amp, copilot, cursor, qwen`. NOTE: this REORDERS v0.2's tail (amp before copilot) per the "alphabetical after the founding four" rule already published in CONTRIBUTING.

**qwen** — bin `qwen`, name `qwen`. Qwen Code is a gemini-cli fork, so flags mirror gemini: LaunchArgs `["-i", prompt]`, HeadlessArgs `["-p", prompt]`. InstallHint `npm install -g @qwen-code/qwen-code`. Window 24h (daily free quota). Strengths: debug .70, feature .75, refactor .75, review .65, docs .75, test .70, other .70. Patterns: `quota`, `rate limit`, `429`, `resource_exhausted`. Fixture per Task convention.

**amp** — bin `amp`, name `amp`. LaunchArgs `[prompt]`, HeadlessArgs `["-x", prompt]`. InstallHint `npm install -g @sourcegraph/amp`. Window 5h. Strengths: debug .80, feature .80, refactor .80, review .80, docs .70, test .80, other .75. Patterns: `rate limit`, `quota`, `usage limit`, `out of free credits`.

Both flag sets are best-guess pending the implementation-time `--help` reality check (CLIs likely absent on this machine — document honestly, v0.2 precedent).

## 6. Launch-event accuracy

An attempt that fails to start (LookPath/exec error) records NO launch event. Ordering becomes: run → on start-error: skip recording, continue; on started: RecordLaunch(+Save) then quota logic. Applies to `cmdHeadless` and `launchWithFallback`.

## Testing

- launch package: timeout/stdin/capture unit tests (real subprocesses via `sh`, Windows-skipped like v0.2).
- E2E: `--json` output parses and matches the envelope schema incl. `attempts` after a failover; `--timeout 1s` against a sleeping fake exits 124 with no failover; piped-stdin failover delivers stdin to the SECOND agent; `--json`/`--timeout` without `-p` error.
- All v0.1/v0.2 suites stay green.
